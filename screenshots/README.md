## 🎨 截图规范

### 设备要求
- **推荐设备**：iPhone 6/7/8 (375x667 pt)
- **分辨率**：750x1334 px (@2x)
- **备选设备**：iPhone X/11 (375x812 pt, 750x1624 px)

### 内容要求
1. **清晰度**：确保截图清晰，文字可读
2. **完整性**：包含完整的页面内容，避免截断
3. **真实性**：使用真实的测试数据，避免空白页面
4. **美观性**：选择视觉效果好的页面状态

### 数据要求
- 使用有意义的测试数据
- 商品名称：使用真实的快餐名称（汉堡、炸鸡、薯条等）
- 价格：合理的价格范围（10-50元）
- 图片：使用高质量的商品图片
- 避免敏感信息：不要包含真实的手机号、地址等

### 状态选择
- **首页**：展示有商品的状态
- **订单列表**：展示多个不同状态的订单
- **购物车**：展示已添加商品的状态
- **个人中心**：展示已登录状态

## 📱 如何截图

### 微信开发者工具截图

1. 打开微信开发者工具
2. 运行小程序项目
3. 调整模拟器尺寸为 iPhone 6/7/8
4. 导航到目标页面
5. 点击工具栏的"截图"按钮，或使用快捷键
6. 保存截图到本目录

### 真机截图

1. 将小程序上传为体验版
2. 使用 iPhone 扫码预览
3. 导航到目标页面
4. 使用 iPhone 截图功能（电源键 + 音量上键）
5. 传输截图到电脑
6. 使用图片编辑工具调整尺寸（如需要）

## 🖼️ 图片处理

### 推荐工具
- **macOS**：预览（Preview）、Sketch
- **Windows**：画图、Photoshop
- **在线工具**：TinyPNG（压缩）、Squoosh

### 处理步骤
1. **裁剪**：去除状态栏（可选）
2. **压缩**：使用 TinyPNG 压缩，保持清晰度
3. **命名**：使用有意义的文件名（如：order.png）
4. **格式**：统一使用 PNG 格式

### 文件大小
- 单张截图建议不超过 500KB
- 如果文件过大，使用压缩工具优化
